# Модуль обработки откачек (PumpingTestProcessing)

Модуль для создания и обработки журналов опытно-фильтрационных работ (ОФР) с графоаналитическим анализом данных.

## Структура папки

```
PumpingTestProcessing/
├── index.js                    # Главный экран со списком проектов
├── NewWizard.js               # Мастер создания/редактирования проектов
├── DataProcessing.js          # Экран обработки данных и построения графиков
├── components/                # Переиспользуемые компоненты
│   ├── DesmosStyleChart.js    # График в стиле Desmos/GeoGebra
│   └── UnifiedMeasurementsModal.js # Единое окно ввода измерений
└── utils/                     # Утилиты и константы
    ├── constants.js           # Константы модуля
    └── helpers.js             # Вспомогательные функции
```

## Основные возможности

### 1. Управление проектами (index.js)
- Создание, редактирование и удаление проектов
- Импорт/экспорт проектов в JSON
- Избранные проекты
- Поиск по проектам

### 2. Мастер создания проектов (NewWizard.js)
- Пошаговое создание журнала откачки
- Ввод основных параметров (тип водоносного горизонта, расход, мощность)
- Настройка типов обработки (откачка/восстановление)
- Управление наблюдательными скважинами
- Ввод расстояний между скважинами

### 3. Обработка данных (DataProcessing.js)
- Построение графиков различных функций (s-t, s-lg(t), s-√t и др.)
- Графоаналитический метод по двум точкам
- Поддержка множественных скважин с цветовой дифференциацией
- Экспорт графиков в галерею

### 4. Компоненты

#### UnifiedMeasurementsModal.js
Оптимизированное окно для ввода всех измерений:
- Единый интерфейс для опытной и наблюдательных скважин
- Виртуализация списка через FlashList
- Дебаунсинг для оптимизации производительности
- Поиск по скважинам
- Переключение между откачкой и восстановлением

## Оптимизация производительности

1. **Мемоизация компонентов**: Использование React.memo для предотвращения лишних ререндеров
2. **Виртуализация списков**: FlashList для больших объемов данных
3. **Дебаунсинг**: Отложенное сохранение при вводе данных
4. **Ограничение данных**: Максимум 500 точек на скважину
5. **Оптимизированные вычисления**: useMemo для тяжелых вычислений

## Используемые библиотеки

- `react-native-svg`: Для рендеринга графиков
- `@shopify/flash-list`: Виртуализация списков
- `react-native-paper`: UI компоненты Material Design
- `@react-native-community/datetimepicker`: Выбор даты/времени
- `expo-media-library`: Сохранение графиков

## Темы и локализация

Модуль полностью поддерживает:
- Светлую и темную темы через ThemeContext
- Русский и английский языки через i18n
- Адаптивные цвета Material You

## Формат данных проекта

```javascript
{
  id: string,
  name: string,
  createdAt: ISO date string,
  lastAccessDate: ISO date string,
  favorite: boolean,
  testType: string,
  layerType: string,
  wellName: string,
  flowRate: string,
  flowRateUnit: string,
  aquiferThickness: string,
  observationWells: Array<{id, name, active}>,
  measurements: {
    [wellId]: {
      pumping: Array<{id, time, timeUnit, drawdown, date}>,
      recovery: Array<{id, time, timeUnit, drawdown, date}>
    }
  },
  distances: {
    [wellId]: string
  },
  dataRows: Array // Преобразованные данные для графика
}
```
